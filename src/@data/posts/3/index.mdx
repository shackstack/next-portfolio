## 문제 상황 1: 서비스 중단 및 저장 공간 제한 문제

- EC2 인스턴스를 사용하는 기존 배포 방식은 애플리케이션의 복잡도 증가에 따라 제한된 저장 공간과 증가하는 비용 문제를 야기했습니다.
- 특히, 정적 리소스의 저장 및 관리에 있어 비효율성이 드러났습니다.

## 해결 과정 1: 정적 리소스 관리 및 CDN 구축

- Amazon S3를 이용하여 정적 리소스(이미지, 스크립트 파일 등)를 관리하고, CloudFront와 함께 CDN을 구축함으로써 이 문제를 해결했습니다.
- 이 방식은 저장 공간 비용을 크게 절감할 뿐만 아니라, 리소스를 엣지 로케이션으로 분산시켜 서비스의 가용성과 성능을 향상시켰습니다.
- 또한, CloudFront CDN은 사용자에게 더 빠른 콘텐츠 전송을 가능하게 하여 사용자 경험을 개선하는 데 기여했습니다.

## 문제 상황 2: 비효율적인 수동 배포 프로세스 및 접근 제한

- 수동으로 S3에 빌드 파일을 드래그 앤 드롭하는 방식은 시간이 많이 소요되었고, 잠재적으로 위험한 작업이었습니다.
- 또한, 교육 기관에서 관리하는 AWS Access Key에 접근하지 못하여, Github-Action을 사용한 자동 배포에 어려움을 겪었습니다.

## 해결 과정 2: EC2를 이용한 자동배포 파이프라인 구축

- 이 문제를 해결하기 위해, EC2 인스턴스에 AWS CLI를 설치하고, Access Key 없이도 S3에 빌드 파일을 자동으로 배포하는 파이프라인을 구축했습니다.
- 이 방식은 배포 과정을 크게 단순화하며, 배포의 안정성과 생산성을 모두 향상시켰습니다.
